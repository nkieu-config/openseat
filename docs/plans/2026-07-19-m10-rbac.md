# M10 — Event Team RBAC Implementation Plan

> **For agentic workers:** execute task-by-task in order; steps use checkbox (`- [ ]`) syntax for tracking. Every task ends green and committed before the next starts.

**Goal:** An event's owner staffs their event by email; every management surface asks *what is your role here* instead of *are you the one account that created this* — per `docs/specs/2026-07-19-m10-rbac-design.md`.

**Architecture:** One policy module owns a strictly-nested role ladder (`owner > manager > staff`) and the denial semantics (outside the team 404, inside 403). Membership is a new table keyed by lowercase email, linking lazily to users; the owner is never a row — ownership derives from `Event.organizerId` at resolution time. Authorization is resolved per request from the database, never from token claims, so revocation is immediate. Every existing `ownedEvent` copy and inline `organizerId` where-clause is deleted in favor of two calls on the policy service.

**Tech Stack:** No new dependencies. One Prisma migration. REST mutations + GraphQL reads per ADR 0006.

## Global Constraints

- **No code comments** anywhere (repo rule; reasoning goes in ADR 0012).
- Conventional Commits, English code/docs/commits, **no AI attribution**.
- Ownership-era behavior is the identity conversion: the owner path must behave byte-for-byte as before, proven by every pre-M10 suite staying green **unedited**.
- Denials: non-member → `NotFoundException('Event not found')` (unchanged string); insufficient role → `ForbiddenException('Your role does not allow this')`.
- After any controller/DTO change: `pnpm --filter api openapi:dump` then `pnpm --filter @openseat/contracts build`.
- Existing gates stay green throughout: `pnpm turbo run lint typecheck build test`, 55 jest e2e, 8 browser journeys, Go suites.
- Every task ends deployable; `APP_VERSION` → `m10` at close.

**Interfaces:**

```ts
const ROLE_RANK = { staff: 1, manager: 2, owner: 3 } as const;
export type EventRoleName = keyof typeof ROLE_RANK;

@Injectable()
export class AccessService {
  constructor(private readonly prisma: PrismaService) {}

  async membershipRole(eventId: string, userId: string): Promise<EventRoleName | null> {
    const member = await this.prisma.teamMember.findFirst({
      where: { eventId, userId },
      select: { role: true },
    });
    return member?.role ?? null;
  }

  async resolveRole(eventId: string, userId: string) {
    const event = await this.prisma.event.findUnique({ where: { id: eventId } });
    if (!event) {
      throw new NotFoundException('Event not found');
    }
    const role =
      event.organizerId === userId
        ? ('owner' as const)
        : await this.membershipRole(eventId, userId);
    return { event, role };
  }

  async requireEventRole(eventId: string, userId: string, minRole: EventRoleName) {
    const { event, role } = await this.resolveRole(eventId, userId);
    if (!role) {
      throw new NotFoundException('Event not found');
    }
    if (ROLE_RANK[role] < ROLE_RANK[minRole]) {
      throw new ForbiddenException('Your role does not allow this');
    }
    return { event, role };
  }
}
```

`AccessModule` exports `AccessService`, imports nothing but Prisma. `TeamModule` imports `AccessModule`.

Team REST (owner-gated via `access.requireEventRole(eventId, userId, 'owner')`):
- `POST /api/events/:eventId/team` body `{ email, role }` (`@IsEmail`, `@IsIn(['manager','staff'])`) → 201 member. Email lowercased; matching the organizer's email → 400 `'The owner is already on the team'`; existing member → 409. When a user with that email exists, set `userId` + `linkedAt` at insert.
- `PATCH /api/events/:eventId/team/:memberId` body `{ role }` → 200; unknown memberId in this event → 404.
- `GET /api/events/:eventId/team` → member rows `{ id, email, role, linked, displayName, createdAt }` (`linked = userId !== null`; `displayName` from the linked user, null when pending). REST rather than GraphQL keeps the typed client the panel uses in one place; the read is owner-only either way.
- `DELETE /api/events/:eventId/team/:memberId` → 204.

Lazy link in `auth.service.ts` — one private method, called after `user.create` in `register` and in `findOrCreateGoogleUser`:

```ts
private async claimMemberships(userId: string, email: string): Promise<void> {
  await this.prisma.teamMember.updateMany({
    where: { email, userId: null },
    data: { userId, linkedAt: new Date() },
  });
}
```
